# Briefing — 2026-07-09

> [!star] Today's must-reads

> - **[Rewriting Bun in Rust](https://bun.com/blog/bun-in-rust)** — the most significant agentic engineering case study yet · Jarred Sumner rewrote Bun (1.3M lines) from Zig to Rust in 11 days using AI agents (Mythos/Fable), with $165K in API tokens. The TypeScript test suite acted as a conformance suite; adversarial review and process-level fixes replaced hand-editing generated code. It shipped in Claude Code on June 17 — nobody noticed. "Boring is good." This redefines what's possible when language choice stops being a one-way decision.

> - **[Introducing GPT‑Live](https://openai.com/index/introducing-gpt-live/)** — OpenAI finally gave voice mode a frontier brain · GPT‑Live replaces the aging GPT-4o-era voice model with a system that delegates to GPT‑5.5 in the background for web search, reasoning, and complex tasks — while keeping the conversation flowing. Simon Willison's review: an hour-long dog walk brainstorming session without frustration. The pattern (conversational frontend + background frontier reasoning) is where AI interfaces are heading.

> - **[TypeScript 7.0](https://devblogs.microsoft.com/typescript/announcing-typescript-7-0/)** — the Go-rewritten compiler is ~10x faster than 6.0 · A landmark release for the JavaScript ecosystem. The compiler rewrite in Go delivers dramatic build speed improvements and sets the stage for the next decade of TypeScript. The architectural decision to switch from TypeScript to Go for the compiler itself is worth studying.

> - **[GitLost: We Tricked GitHub's AI Agent into Leaking Private Repos](https://noma.security/blog/gitlost-how-we-tricked-githubs-ai-agent-into-leaking-private-repos/)** — the SQL-injection moment for agentic AI · Noma Security systematically exploited GitHub's AI coding agent to exfiltrate private repository contents. Prompt injection attacks on AI agents are becoming a category-wide threat — this is the definitive case study showing why agent security needs to be treated as seriously as web app security.

> - **[Experiences with local models for coding](https://martinfowler.com/articles/exploring-gen-ai/local-models-for-coding-experiences.html)** — can you ditch the API for local LLMs? · Birgitta Böckeler (Martin Fowler's team) ran a two-part rigorous evaluation: comparing local models on standard coding tasks, then using the best one for daily work. Practical, measured, and relevant to every engineer wondering whether local models are viable yet.
