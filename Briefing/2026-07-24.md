> - **[GitLost: Indirect Prompt Injection Exploits GitHub's AI Agent to Leak Private Repository Data](https://noma.security/blog/gitlost-how-we-tricked-githubs-ai-agent-into-leaking-private-repos/)** (InfoQ) — prompt injection is the new SQL injection · Noma Security embedded concealed instructions in a public GitHub issue. The word "Additionally" bypassed GitHub's guardrails, causing the agent to read a private repo and post its contents publicly. The finding crystallizes a hard truth: trust boundaries in agentic systems are enforced by model behavior, not code. If an agent has access to your private repos, treat everything as one well-crafted issue away from public.
>
> - **[Why Software Factories Fail (or: harness engineering is not enough)](https://github.com/humanlayer/advanced-context-engineering-for-coding-agents/blob/main/wsff.md)** (181pts HN) — the most thorough rebuttal to "just ship more loops" yet · Dex Horthy (HumanLayer) traces the software factory idea from a 1968 NATO conference through today's lights-off factories, then systematically argues: no amount of harness engineering or loopsmaxxing can fix what is fundamentally a model-training problem. PR review quality is down, incidents are up, and codebases are falling apart faster than ever. The post is long; the thinking is careful.
>
> - **[A Beginner's Guide to Clocks, Causality, and Ordering in Distributed Systems](https://blog.bytebytego.com/p/a-beginners-guide-to-clocks-causality)** (ByteByteGo, Jul 23) — why reading the time is a hard problem · Lamport clocks, vector clocks, causal ordering — ByteByteGo distills one of distributed systems' most notoriously difficult topics into clear, visual explanations. If you build anything that spans more than one machine, understanding causality is non-negotiable.

> - **[Software rendering in 500 lines of bare C++](https://ssloy.github.io/tinyrenderer/)** (244pts HN, Lobsters) — the kind of thing that reminds you why you got into this field · A tiny, elegant software renderer that builds from drawing a line to full 3D rendering in a few hundred lines of readable C++. Both HN and Lobsters loved it. Sometimes the best way forward is to go back to fundamentals.

> - **[Not just development, distribution of software may change as well](https://antirez.com/news/170)** (antirez) — Redis's creator on what AI coding means for how we ship software · When the user can modify your code as easily as you can, the stable/unstable branch model breaks down. Experimental branches become first-class artifacts. Documentation needs to be written for coding agents, not just humans. Antirez argues repositories should be "templates for how to do things" rather than finished products. Short, original, worth your 5 minutes.

> - **[Laguna S 2.1: Cheaper than DeepSeek v4 Flash, Better than V4 Pro](https://www.latent.space/p/ainews-laguna-s-21-released-cheaper)** (Latent Space) — a Western neolab proves you don't need 1T+ parameters to compete · Poolside AI's 118B MOE beats Thinking Machines' ~1T open-weights model. > - **[DARPA, US Air Force fly AI-controlled F-16](https://www.darpa.mil/news/2026/darpa-us-air-force-fly-ai-controlled-f-16)** (183pts HN) — an AI agent flew a real fighter jet in combat scenarios · The engineering challenges — real-time control, safety guarantees, adversarial robustness — are the same ones civilian AI engineers need to think about.
