### AG201 · U1: The Transformer — Your Agent's Substrate

**Outcome:** u1.transformer (easy · completion)

Carried from yesterday. This is the foundation — agents run on LLMs, and every design decision you'll make for the next 5 months is constrained by what happens inside the model. Master the substrate first.

**Concept — the mental model.**

A transformer processes a sequence of tokens. Every token "attends" to every other token through three learned projections: **Query** (what am I looking for?), **Key** (what do I contain?), **Value** (what do I contribute if you attend to me?). The dot-product of Q and K tells each token where to look; softmax turns that into weights; multiplying by V produces the attended output. This is *self-attention* — the engine.

**Multi-head** runs several of these in parallel (different subspaces), concatenates, and projects — so the model can attend to syntax, semantics, and position simultaneously. **Residual connections** add the input back to the output of every sublayer (attention + FFN), then **layer norm** stabilizes. Without residuals, deep transformers don't train. **Positional encoding** injects order because attention is permutation-invariant — without it, "dog bites man" and "man bites dog" look identical.

During **inference**, the model generates one token at a time (autoregressive). The KV cache stores already-computed Key and Value projections so you don't recompute them — this is why context windows are expensive (quadratic memory) and why context engineering matters.

**Decoder-only** means no separate encoder — the model reads and generates from the same stack with causal (look-back-only) masking. This design scales better than encoder-decoder, produces more general representations, and GPT-4/Claude/DeepSeek all use it. The **Bitter Lesson** (Sutton, 2019): general methods that scale with compute beat hand-crafted domain knowledge. Decoder-only won because it scales, not because it's conceptually cleaner.

**Why this matters for agents:** your agent sends prompts and tools into a context window. Every design decision — how you structure the system message, when you compact history, how much you offload to external memory — is constrained by attention economics (KV-cache cost) and context-window limits.
